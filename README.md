# 🌐 Packet Tracer - Observe Traffic Flow in a Routed Network




## 📌 Overview
This project demonstrates how network traffic behaves in both unrouted and routed environments using Cisco Packet Tracer.

The lab highlights how implementing routing between LANs improves network efficiency, reduces broadcast traffic, and enhances scalability.




---




## 🎯 Objectives




  
### Part 1: Unrouted LAN
- Observe traffic flow in a single network
- Analyze ARP broadcasts across the entire LAN




### Part 2: Network Reconfiguration
- Connect switches directly to router interfaces
- Assign separate IPv4 networks to each department
- Configure hosts using DHCP




### Part 3: Routed Network
- Analyze traffic flow after segmentation
- Observe reduced broadcast domains
- Understand router role in traffic control




---





## 🧪 Tools Used
- Cisco Packet Tracer
- Simulation Mode (PDU Analysis)
- ARP & ICMP Protocol Analysis




---




## 🔍 Key Observations






### 🔴 Unrouted Network
- All devices receive ARP broadcasts
- High network congestion
- Reduced efficiency due to single broadcast domain




### 🟢 Routed Network
- ARP requests are limited within each subnet
- Reduced unnecessary traffic
- Improved performance and scalability



---




## 📡 Key Concepts





- ARP (Address Resolution Protocol)
- Broadcast vs Unicast traffic
- Subnetting and network segmentation
- Role of routers in controlling traffic flow




---




## 💡 Key Learnings




- Network segmentation improves efficiency
- Routers reduce broadcast traffic
- Large networks must be divided into subnets
- Proper design is critical for scalability
